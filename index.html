Responsive layouts don't have to be a struggle

It's me, Saad. My dream is to be a web developer, and I love what I do and learning and growing. Huge thanks to the Discord community of Kevin Powell, who helps me in every part of my journey.

Mobile-first

The best thing I learned in this course is that Kevin Powell (the YouTuber) says that in most conditions, we should always go for the mobile-first approach and make the layouts for smaller devices first.

Efficient

This way is really efficient because we can just make the layouts for mobiles first and then move on to larger devices like min-width (700px) and write the CSS for the respective screen size.

Done right

In this way, we won't face problems like people often struggle with for smaller device layouts. By starting with smaller devices, we minimize problems and errors. That's why this is the best approach.

It doesn't have to be so hard

The main thing Kevin talks about in this course, which I also really like, is that all the HTML and CSS we write is inherently responsive; it is us who make it unresponsive. For example, plain HTML is responsive, right? It's the developer who does something that makes it unresponsive. According to me, this is the best insight Kevin shares in this course.

Another thing I learned in this course is to use the correct sizing units, like when to use %, em, and rem. That's the magic of responsive layouts; it's just that easy. Another important tip is to always set the width, like max-width. For example, if I give something 90vw, it will be 90vw on smaller devices, but if we set a max-width to it, it can also work great on larger screens.

Break it down

Another thing I learned is to plan everything in advance. When looking at a layout, I used to just hop on and start writing without planning it in my mind or figuring out how I would write it.

Work your way up

To be a successful developer, it's all about working my way up, making projects, learning new things, and enjoying the journey. I love everything I do, and I am always learning new techniques every day.

It's easy

It's easy if you understand the concepts, make projects, and learn from your mistakes. No matter how small a project is, there is always something to learn. For example, in this project, I didn't use as many div.
